======================================================
BRAINSTORM DESKTOP APP (ELECTRON) - DEVELOPER GUIDE
======================================================

This guide explains how your web application is turned into a Windows Desktop App (.exe) and how to manage it.

------------------------------------------------------
1. WHAT DOES "BUILDING" MEAN?
------------------------------------------------------
"Building" is the process of translating your human-readable source code (React, TypeScript) into an optimized, bundled format that computers and browsers can execute efficiently. 

For the Desktop App, "building" does two things:
1. It bundles your web code (React) into static HTML/CSS/JS files.
2. It wraps those static files inside "Electron"—a framework that packages web apps into standalone desktop windows. It then compiles this wrapper into a final `.exe` installer (or `.zip` file) that users can download and double-click to install on their Windows machines!

------------------------------------------------------
2. HOW TO BUILD THE DESKTOP APP
------------------------------------------------------
Whenever you make changes to the codebase (like changing colors, adding features, or fixing bugs) and you want those changes to appear in the Desktop App, you MUST run a build.

To build the app:
1. Open a new terminal in your project folder (`C:\Users\ammar\Downloads\brainstorm (1)`).
2. Run this exact command:
   
   npm run desktop:build

3. Wait 1-3 minutes. It will first build the Vite React app, and then `electron-builder` will package everything.
4. Once finished, you will find your compiled executable files inside the `dist-electron` folder!

------------------------------------------------------
3. HOW THE DOWNLOAD BUTTON WORKS
------------------------------------------------------
If you are running the app locally using `npm run dev` and testing it on `localhost:5173`, the "Download App (ZIP)" button **will** perfectly work right now! 

However, your actual *live website* that the public visits (e.g. hosted on Vercel or Netlify) does not know about the file we just created on your computer.

HOW TO UPDATE YOUR LIVE WEBSITE DOWNLOAD BUTTON:
1. Every time you want to offer a new version, run `npm run desktop:build`.
2. Locate the generated executable or zip and place it inside your `public/` folder.
3. Push your entire codebase (including the .zip/.exe inside the public folder) to your Git repository holding the website code (e.g., GitHub).
4. Wait for your web host (Vercel/Netlify) to finish deploying the new code.
5. Once your live website redeploys, clicking the "Download App" button on the live internet will successfully download the file!

------------------------------------------------------
4. HOW TO LAUNCH AN UPDATE TO EXISTING USERS
------------------------------------------------------
Your desktop app has Auto-Updating built in! If a user has already downloaded and installed Brainstorm, you can push updates directly to their computer when you make changes to the code.

Here is the exact step-by-step process to release an Update:

1. **Change the Version Number:**
   Open your `package.json` file. Find the `"version": "0.0.0"` line at the top. Change it to structurally reflect an update (e.g., `"version": "1.0.1"`). 

2. **Build the Installer:**
   Run `npm run desktop:build` in your terminal. This will package all your new code into a brand new `Brainstorm-Setup-1.0.1.exe` file located in the `dist-electron` folder.

3. **Create a GitHub Release:**
   - Go to your application's GitHub repository page: `https://github.com/ammarsapru/brainstorm`
   - Click on "Releases" on the right sidebar, then click "Draft a new release".
   - Click "Choose a tag" and type the exact version starting with a 'v' (e.g., `v1.0.1`) and select "Create new tag".
   - Give the release a Title (like "Performance Updates") and type notes about what changed.
   - **Crucial Step:** Drag and drop the `Brainstorm-Setup-1.0.1.exe` file you built into the "Attach binaries" box at the bottom.
   
4. **Publish!**
   Click "Publish release". 

**What happens next?**
The next time your users open the Brainstorm app on their computer, it will automatically connect to your Github, detect that `v1.0.1` exists, download the new `.exe` file in the background, and ask them "An update is ready, would you like to install it and restart?". 

Your users get the new features instantly without having to visit your website!
