## Fix Cursor Jumping and Enter Key Logic
**What you asked for:**
You reported that clicking on a line moved the cursor to the start immediately, and pressing Enter did not create a new line. Later, you clarified that Backspace also caused the cursor to jump.

**What I did:**
- I refactored the `DocumentEditor` to use a `ContentBlock` component.
- I fixed the cursor jumping issue by implementing `useLayoutEffect` to manually manage `innerHTML` updates, preventing React from resetting the cursor on every render.
- I fixed the Enter key by using `useRef` to ensure event handlers always have access to the latest state, preventing stale closures.

## Distinguish Enter vs. Shift+Enter & Fix Selection
**What you asked for:**
You requested a clear distinction between "T-Lines" (blocks) and "L-Lines" (visual lines). Specifically, `Enter` should create a new block, and `Shift+Enter` should create a new line within the same block. You also noted that selecting text in the document editor blindly selected the card behind it.

**What I did:**
- I updated the `handleKeyDown` logic to perfectly match this specification.
- `Enter` now rigorously splits the current block into two.
- `Shift+Enter` allows the default behavior (inserting a line break) to occur within the block.
- I added `e.stopPropagation()` to the editor's container events (`onClick`, `onMouseDown`, `onMouseUp`) to strictly isolate editor interactions from the canvas background.

## Fix "SPLIT" Text Regression
**What you asked for:**
You pointed out that pressing Enter was leaving visible `__SPLIT__` text in the document.

**What I did:**
- I identified that `element.normalize()` was merging my text marker.
- I replaced the text marker with a temporary, invisible `<span>` element and removed the normalization step, ensuring clean splits without visual artifacts.

## Polished Drag and Drop
**What you asked for:**
You reported that dragging lines caused unbearable flickering because they were swapping positions instantly.

**What I did:**
- I implemented a "Drop Indicator" system.
- Instead of live-reordering the list (which caused the layout shift/flicker), I added a visual blue line to indicate the drop target.
- The actual reordering now only happens when you release the mouse button (`onDrop`), resulting in a smooth, stable experience.
